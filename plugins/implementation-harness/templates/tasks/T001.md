## Overview

Lay the minimal project skeleton and turn the Definition-of-Done tooling on. This is the ONE task that
defines the project-wide conventions (formatter, linter, test runner, CI) that gate every later build —
and its whole-repo tooling sweeps a tree that already vendors files the loop must never touch — so a human
sets it up once, by hand, rather than the loop. (`gate: needs-human`: the loop skips this task and records
`failed:blocked` until you mark it done.)

## Do

1. **Create a real, empty build.** Add the project manifest + a trivial entrypoint for your stack so
   there's something to build (e.g. `package.json` + `src/index.js`; `Cargo.toml` + `src/main.rs`;
   `pyproject.toml` + a package; `go.mod` + `main.go`).
2. **Wire the real DoD into CI.** Put your exact format / lint / test / build commands into
   `.github/workflows/ci.yml` — they must match `.harness/docs/HARNESS.md` §5 and
   `.harness/config/harness.env`.
3. **Exclude the harness from your own tooling (critical).** Your format/lint/test commands run over the
   whole repo, which now vendors `.harness/**` (hand-written `dashboard/*.js`, docs, JSON) and the
   `CLAUDE.md` / `README.md` prose the scaffolder wrote — none of it meant to satisfy your formatter. Add
   the exclusions your toolchain uses, e.g.:
   - a `.prettierignore` containing `.harness/` and `CLAUDE.md` (the scaffolder may already have shipped this);
   - `.harness/**` in the eslint flat-config `ignores`;
   - a test path/glob that does not pick up `.harness/dashboard/*.test.js` (`node --test`/jest recurse by default).

   Python/Go usually need nothing here — no `.py`/`.go` lives under `.harness/`.
4. **Get it green** — format / lint / test / build pass locally, then CI passes on `main`.

## Done when

CI is green on `main`; the Definition-of-Done commands in `.harness/docs/HARNESS.md` §5 match exactly what
CI runs; and a whole-repo `format --check` (and lint/test) is clean — i.e. the tooling ignores `.harness/**`
and the harness-authored root prose. Then mark this task done so its dependents unblock:
`.harness/scripts/mark-done.sh T001` (or the dashboard's "Mark done").
